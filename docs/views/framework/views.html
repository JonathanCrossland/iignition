Views

Views are the building blocks of your iignition application, representing individual pages or content sections that can be loaded into containers.

Views Folder

The views folder is where you organize your application's content. Here's a typical structure:

your-project/
├── views/
│   ├── home.html
│   ├── about.html
│   ├── contact.html
│   ├── components/
│   │   ├── accordion.html
│   │   ├── datatable.html
│   │   └── ...
│   └── framework/
│       ├── installation.html
│       ├── containers.html
│       └── views.html

View File Format

Each view file should be a complete HTML fragment that can be loaded into a container. Example:

<div class="view-content">
    <h1>About Us</h1>
    <p>Welcome to our company...</p>
</div>

<style>
    .view-content {
        padding: var(--spacing-lg);
    }
</style>

Basic View Loading

Use the $i.show() method to load views into containers:

// Load a view into the default container
$i.show('#!views/home.html');

// Load a view into a specific container
$i.show('#!views/about.html', 'main');

// Load a view into multiple containers
$i.show('#!views/dashboard.html', ['main', 'sidebar']);

View Loading Options

Option Description
url The path to the view file (required)
container Container ID or array of container IDs (optional)
options Additional loading options (optional)

View Lifecycle Events

You can listen for view-related events:

// View loaded successfully
$i.on('view:loaded', (view, container) => {
    console.log(`View ${view} loaded into ${container}`);
});

// View loading failed
$i.on('view:error', (view, error) => {
    console.error(`Error loading view ${view}:`, error);
});

// View unloaded
$i.on('view:unloaded', (view, container) => {
    console.log(`View ${view} unloaded from ${container}`);
});

View Organization

  • Keep views modular and focused on a single purpose
  • Use consistent naming conventions for view files
  • Organize views into logical subfolders

Performance Tips

  • Minimize the size of view files
  • Use lazy loading for complex views
  • Cache frequently accessed views

Overview

Dockable views allow you to create resizable, draggable, and stackable window-like interfaces in your application. They are perfect for building complex layouts like IDEs, dashboards, or multi-panel applications.

Basic Usage

<dockable-view>
    <div class="dockable-view-window">
        <div class="dockable-view-window-header">
            <div class="dockable-view-window-title">Window Title</div>
            <div class="dockable-view-window-controls">
                <button class="dockable-view-maximize">□</button>
                <button class="dockable-view-close">×</button>
            </div>
        </div>
        <div class="dockable-view-window-content">
            Your content here
        </div>
    </div>
</dockable-view>

Attributes

Attribute Description
locked When set to "true", prevents window dragging and resizing
allow-stacking When set to "true", allows windows to be stacked on top of each other
title Sets the window title

Features

  • Draggable Windows: Click and drag window headers to move them
  • Resizable: Drag the splitter between windows to resize them
  • Window Stacking: Stack multiple windows and switch between them using tabs
  • Maximization: Windows can be maximized to fill the entire container
  • Customizable Styling: Uses CSS variables for easy theming

CSS Variables

Variable Description
--dockable-window-bg Window background color
--dockable-window-border Window border color
--dockable-header-bg Window header background color
--dockable-header-border Window header border color
--dockable-text-color Text color for window title and controls
--dockable-splitter-color Color of the resize splitter
--dockable-splitter-size Width of the resize splitter
--dockable-content-padding Padding inside window content area

Best Practices

  • Use meaningful titles for each window to help users identify content
  • Consider using the locked attribute for windows that should maintain their position
  • Implement a logical stacking order for related windows
  • Provide sufficient padding in window content for better readability
  • Use consistent styling across all dockable views in your application
