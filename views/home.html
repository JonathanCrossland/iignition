Follow W3C Rules

If it's not valid HTML, or CSS, it's not valid syntax.

Features should embed and fit within existing HTML and CSS.

The more compatible it is, with less custom elements, the more reusable your code is and the less reliant on us you are.


Low Learning Curve

Always provide the simpler option, and the one that supplements existing knowledge.

There are other libraries providing bells and thistles, we want to provide a minimal, consistent and standards based alternative.


Provide Good Value

It's pointless copying existing libraries, providing Yet-another-js-library, that looks the same.

We want high value on it's own merits, steered by standards and non obtrusive design.

Features must offer good value to learning curve ratio
