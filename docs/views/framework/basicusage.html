Basic Usage

Learn how to use iignition's core features and components in your web applications.

View Containers

View containers are the foundation of iignition applications. They allow you to:

  • Load content dynamically into specific areas
  • Create single-page application behavior
  • Manage content state and transitions
<div data-viewcontainer="main"></div>

<script>
    // Load content into the container
    $i.show('#!views/content.html', 'main');
</script>

Navigation

iignition provides two ways to navigate between views:

  • Using hash-based URLs with #! prefix
  • Using the data-link attribute on elements
<!-- Hash-based navigation -->
<a href="#!views/page.html">Go to Page</a>

<!-- Data attribute navigation -->
<button data-link="#!views/page.html">Go to Page</button>

Data Handling

You can pass data between views using:

  • URL parameters
  • State management
  • View events
// Pass data via URL
$i.show('#!views/page.html?id=123', 'main');

// Pass data via state
$i.show('#!views/page.html', 'main', { id: 123 });

Component Integration

iignition components are web components that can be used directly in your HTML:

<!-- Using an accordion component -->
<accordion-component title="My Section">
    <div slot="content">
        Content goes here
    </div>
</accordion-component>

<!-- Using a modal dialog -->
<modal-dialog title="My Dialog">
    <div slot="content">
        Dialog content
    </div>
</modal-dialog>

Component Customization

Components can be customized using:

  • CSS variables for styling
  • Attributes for behavior
  • Events for interactivity
/* Customize component appearance */
my-component {
    --component-bg: #ffffff;
    --component-color: #000000;
}

project/
├── js/
│   ├── ii.js
│   └── ii_components.js
├── css/
│   └── styles.css

Performance Tips

  • Use view containers efficiently
  • Implement proper error handling
  • Cache frequently used views
  • Optimize component usage

Common Patterns

  • Create reusable view templates
  • Implement consistent navigation patterns
  • Use event delegation for dynamic content
  • Follow component lifecycle best practices
