Data Handling

iignition provides a powerful data handling system with a wrapper around the Fetch API for making HTTP requests and managing data.

Basic Usage

The $i.fetch() method provides a simplified interface for making HTTP requests:

// GET request
$i.fetch('/api/users')
    .then(response => console.log(response))
    .catch(error => console.error(error));

// POST request with data
$i.fetch('/api/users', {
    method: 'POST',
    body: JSON.stringify({ name: 'John' })
})
    .then(response => console.log(response))
    .catch(error => console.error(error));

Fetch Options

Option Description Default
method HTTP method (GET, POST, PUT, DELETE, etc.) GET
headers Request headers {}
body Request body (automatically stringified if object) null
timeout Request timeout in milliseconds 30000

Data Store

iignition provides a simple data store for managing application state:

// Set data
$i.data.set('users', [{ id: 1, name: 'John' }]);

// Get data
const users = $i.data.get('users');

// Update data
$i.data.update('users', users => [...users, { id: 2, name: 'Jane' }]);

// Remove data
$i.data.remove('users');

Data Events

Listen for data changes:

// Listen for data changes
$i.data.on('change:users', (newData, oldData) => {
    console.log('Users data changed:', newData);
});

// Listen for all data changes
$i.data.on('change', (key, newData, oldData) => {
    console.log(`${key} changed:`, newData);
});

Data Organization

  • Use meaningful keys for data storage
  • Group related data under common prefixes
  • Keep data structures consistent

Performance Tips

  • Minimize data updates to reduce re-renders
  • Implement proper error handling for fetch requests
  • Use appropriate HTTP methods for different operations
