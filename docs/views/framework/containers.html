View Containers

View containers are a core feature of iignition that enable dynamic content loading and navigation within your application.

View containers allow you to:

  • Dynamically load content into specific areas of your page
  • Create single-page application (SPA) behavior without complex routing
  • Manage content state and transitions
  • Implement deep linking and browser history

HTML Structure

<!-- Define a container in your HTML -->
<div data-viewcontainer="main"></div>

<!-- Load content into the container -->
<script>
    $i.show('#!views/content.html', 'main');
</script>

Container Attributes

Attribute Description
data-viewcontainer Identifies the container for content loading
data-persist Optional. If present, container content persists during navigation

Link Structure

Links in iignition follow this format:

<a href="#!path/to/content.html">Link Text</a>

Link Examples

Link Type Example Description
Basic View #!views/home.html Loads content into the default container
Specific Container #!views/content.html|sidebar Loads content into the 'sidebar' container
Multiple Containers #!views/content.html|main,sidebar Loads content into multiple containers

Container Events

// Listen for container events
$i.on('container:loaded', (container, content) => {
    console.log(`Content loaded into ${container}`);
});

$i.on('container:error', (container, error) => {
    console.error(`Error loading content into ${container}:`, error);
});

Container Methods

Method Description
$i.show(url, container) Loads content from URL into specified container
$i.hide(container) Hides content in specified container
$i.clear(container) Clears content from specified container

Container Organization

  • Use meaningful container names that describe their purpose
  • Keep container structure consistent across your application
  • Use data-persist for containers that should maintain state

Link Management

  • Use relative paths for internal links
  • Include container targets in links when needed
  • Test links in different navigation scenarios
