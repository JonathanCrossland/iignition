Views

Views are the building blocks of your iignition application, representing individual pages or content sections that can be loaded into containers.

Views Folder

The views folder is where you organize your application's content. Here's a typical structure:

your-project/
├── views/
│   ├── home.html
│   ├── about.html
│   ├── contact.html
│   ├── components/
│   │   ├── accordion.html
│   │   ├── datatable.html
│   │   └── ...
│   └── framework/
│       ├── installation.html
│       ├── containers.html
│       └── views.html

View File Format

Each view file should be a complete HTML fragment that can be loaded into a container. Example:

<div class="view-content">
    <h1>About Us</h1>
    <p>Welcome to our company...</p>
</div>

<style>
    .view-content {
        padding: var(--spacing-lg);
    }
</style>

Basic View Loading

Use the $i.show() method to load views into containers:

// Load a view into the default container
$i.show('#!views/home.html');

// Load a view into a specific container
$i.show('#!views/about.html', 'main');

// Load a view into multiple containers
$i.show('#!views/dashboard.html', ['main', 'sidebar']);

View Loading Options

Option Description
url The path to the view file (required)
container Container ID or array of container IDs (optional)
options Additional loading options (optional)

View Lifecycle Events

You can listen for view-related events:

// View loaded successfully
$i.on('view:loaded', (view, container) => {
    console.log(`View ${view} loaded into ${container}`);
});

// View loading failed
$i.on('view:error', (view, error) => {
    console.error(`Error loading view ${view}:`, error);
});

// View unloaded
$i.on('view:unloaded', (view, container) => {
    console.log(`View ${view} unloaded from ${container}`);
});

View Organization

  • Keep views modular and focused on a single purpose
  • Use consistent naming conventions for view files
  • Organize views into logical subfolders

Performance Tips

  • Minimize the size of view files
  • Use lazy loading for complex views
  • Cache frequently accessed views
